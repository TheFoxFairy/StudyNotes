### 进程的同步与互斥

**互斥**：是指散布在不同进程之间的若干程序片段，当某个进程执行其中的一个程序片段时，其他进程就不能运行它们之中的任一程序片段，只能等到该进程运行完之后才可以继续运行。 

**同步**：是指散布在不同进程之间的若干程序片段，它们的运行必须严格按照一定的先后次序来运行，这种次序依赖于要完成的任务。

![image-20210326091941506](https://cdn.jsdelivr.net/gh/TheFoxFairy/ImgStg/202201112004890.png)

### PV操作

PV操作是一种实现进程互斥与同步的有效方法。PV操作与信号量的处理相关，P表示**通过**的意思，V表示**释放**的意思。

* 临界资源：各进程间需要互斥的方式对其进行共享的资源，如打印机、磁带机等
* 临界区：每个进程中访问临界资源的那段代码称为临界区
* 信号量：是一种特殊的变量

![image-20210326092551878](https://cdn.jsdelivr.net/gh/TheFoxFairy/ImgStg/202201112004892.png)

**示例**：生产者与消费者问题

### 死锁问题

进程管理是操作系统的核心，但如果设计不当，就会出现死锁问题。如果一个进程在等待一件不可能发生的事，则进程就死锁了。而如果一个或多个进程产生死锁，就会造成系统死锁。

例：系统有3个进程：A、B、C。这3个进程都需要5个系统资源。如果系统至少有多少个资源，则不可能发生死锁。

![image-20210326101200924](https://cdn.jsdelivr.net/gh/TheFoxFairy/ImgStg/202201112004897.png)

```
避免造成死锁，需要 进程A资源-1 + 进程B资源-1 + 进程C资源-1 = 12
```

## 存储管理

### 分区存储组织

某计算机系统的内存大小为128k，采用可变分区分配方式进行内存分配，当前系统的内存分块情况如下图所示，现有作业4申请内存9k，几种不同的存储分配算法在分配中，会产生什么样的结果呢？

![image-20210326104431033](https://cdn.jsdelivr.net/gh/TheFoxFairy/ImgStg/202201112004900.png)

```
* 首次适应法：第一遇到的适合的区域，进行分配
* 最佳适应法：找到最适合的区域，进行分配
* 最差适应法：找到最大的区域，进行分配
* 循环首次适应法：在分配内存空间时，不再每次从表头（链首）开始查找，而是从上次找到空闲区的下一个空闲开始查找，
		直到找到第一个能满足要求的空闲区为止，并从中划出一块与请求大小相等的内存空间分配给作业。
		该算法能使内存中的空闲区分布得较均匀。
		也就是从作业3开始查找。
```

### 内存管理方式

#### 页式存储

用户程序的地址空间被划分成若干固定大小的区域，称为“页”，相应地，内存空间分成若干个物理块，页和块的大小相等。可将用户程序的任一页放在内存的任一块中，实现了离散分配。

![image-20210326105917416](https://cdn.jsdelivr.net/gh/TheFoxFairy/ImgStg/202201112004901.png)

* 优点：利用率高，碎片少，分配及管理简单
* 缺点：增加了系统开销；可能产生抖动现象

实例：通过逻辑地址查找物理地址

![image-20210326110648658](https://cdn.jsdelivr.net/gh/TheFoxFairy/ImgStg/202201112004902.png)

```
解：
一页大小：4K = 2^12，说明一个页面的地址大小为12位
因此5A29H中A29H为地址，物理块号/页帧号为6，即6A29H

淘汰页号：其他状态位为0的表示不在内存中，状态位为1的表示在内存中，有0125
        其中访问过不能被淘汰，访问位为0的有1，因此淘汰页号1
```

#### 段式存储

页面是主存物理空间中划分出来的等长的固定区域。分页方式的优点是页长固定，因而便于构造页表、易于管理，且不存在外碎片。但分页方式的缺点是页长与程序的逻辑大小不相关。例如，某个时刻一个子程序可能有一部分在主存中，另一部分则在辅存中。这不利于编程时的独立性，并给换入换出处理、存储保护和存储共享等操作造成麻烦。

![image-20210326111418668](https://cdn.jsdelivr.net/gh/TheFoxFairy/ImgStg/202201112004903.png)

* 优点：多道程序**共享**内存，各段程序修改互不影响
* 缺点：内存**利用率低**，内存**碎片浪费大**

> 逻辑地址由段号和段内地址构成

#### 段页式存储

段页式存储组织是分段式和分页式结合的存储组织方法，这样可充分利用分段管理和分页管理的优点。

![image-20210326112717731](https://cdn.jsdelivr.net/gh/TheFoxFairy/ImgStg/202201112004906.png)

* 优点：空间浪费小，存储共享容易，存储保护容易，能动态链接
* 缺点：由于管理软件的增加，复杂性和开销也随之增加，需要的硬件以及占用的内容也有所增加，使得执行大大下降

> 段页式存储管理：先将逻辑空间按段氏管理分成若干段，再把段内空间按页式管理分成若干页

### 快表

快表示一块小容量的相联存储器（Associative Memory），由高速缓冲器组成，速度快，并且可以从硬件上保证按内容并行查找，一般用来存放当前访问最频繁的少数活动页面的页号。

> 慢表是放在内存中的，快表是放在cache中的。

### 页面置换算法

* 最优算法（Optimal,OPT）
* 随机算法（RAND）
* **先进先出算法（FIFIO）**：有可能产生“抖动”。例如，`432143543215`序列，用3个页面，比4个缺页要少
* **最近最少使用算法（LRU）**：不会“抖动”
