## 数据的表示

### 进制的转换

**$R$进制转十进制使用按权展开法**：将$R$进制数的每一位数用$R^k$形式表示，即幂的底数是$R$，指数是$k$，$k$与该位和小数点之间的距离有关。当该位位于小数点左边，$k$值是该位和小数点之间的数码的个数；而当该位位于小数的右边，$k$值是负值，其绝对值是该位和小数点之间数码的个数加1。

* 二进制转十进制

```
10100.01 = 1x2^4 + 0x2^3 + 1x2^2 + 0x2^1 + 0x2^0 + 0x2^(-1) + 0x2^(-2) = 16 + 4 + 0.25 = 20.25
```

* 七进制转十进制

```
604.01 = 6x7^2 + 0x7^1 + 4x7^0 + 0x7^(-1) + 1x7^(-2)
```

**十进制转$R$进制使用短除法**：

```
83(2) => 
	1. 2 | 83 => 1

94(16) =>

94(16) = 5E
```

### 原码、反码、补码、移码

![image-20210318112241356](https://cdn.jsdelivr.net/gh/TheFoxFairy/ImgStg/202201112000311.png)

左边的第一位数表示正负号（符号），其中0表示为正号，1表示为负号。

在原码中，进行1-1的计算，会得到-2，因此提出了其他编码形式：

* 反码在表示正数的时候与原码一样，在表示负数的时候，除了符号位以外，其余按位取反，计算1-1，之后取反，结果为-0。
* 补码在表示正数的时候与原码一样，在表示负数的时候，是在原来的反码的基础上加+1，计算1-1，结果为0。
* 移码是在补码的基础上首位取反

## 流水线技术

### 基本概念

流水线：是指在程序执行时多条指令**重叠**进行操作的一种准**并行**处理实现技术。各种部件**同时处理**是针对不同指令而言的，它们可同时为多条指令的不同部分进行工作，以**提高**各部件得到利用率和指令的平均执行速度。

取一条指令分为：**取指-分析-执行**三个阶段

![image-20210318131526675](https://cdn.jsdelivr.net/gh/TheFoxFairy/ImgStg/202201112000313.png)

### 流水线计算

* 流水线周期为**执行时间最长的一段**
* 流水线计算公式为：**1条指令执行时间 + （指令条数-1）* 流水线周期**

1. 理论公式：$(t_1+t_2+\cdots+t_k)+(n-1)*\Delta t$
2. 实践公式：$(k+n-1)*\Delta t$，其中$k$为一条指令所包含的部分的多少。

![image-20210318141938818](https://cdn.jsdelivr.net/gh/TheFoxFairy/ImgStg/202201112000314.png)

例：若指令流水线把一条指令分为取指、分析和执行三部分，且三部分点的时间分别是取指2ns，分析2ns，执行1ns。那么，流水线周期是多少？100条指令全部执行完毕需要的时间是多少？

```
流水线周期：△t = max(2ns,2ns,1ns) = 2ns
100条指令全部执行完毕需要的时间：(2+2+1)ns + (100-1)*2ns = 5ns + 99*2ns = 5ns +198ns = 203ns (理论)
							(3+100-1)*2ns = 204ns (实践)
```

### 流水线吞吐率计算

流水线的吞吐率`（Though Put rate，TP）`是指在单位时间内流水线所完成的任务数量或输出的结果数量。

计算流水线吞吐率的最基本的公式如下：
$$
TP = \frac{指令条数}{流水线执行时间}
$$
流水线最大吞吐率：
$$
TP_{max} = \lim_{n\rightarrow \infty}\frac{n}{(k+n-1)\Delta t} = \frac{1}{\Delta t}
$$


例：若指令流水线把一条指令分为取指、分析和执行三部分，且三部分点的时间分别是取指2ns，分析2ns，执行1ns。那么，100条指令全部执行完毕，其流水线的吞吐率是多少？流水线的最大吞吐率是多少？

```
流水线的吞吐率：TP = 100 / 203
流水线的最大吞吐率：TP(max) = 1 / △t = 1/2 = 0.5
```

### 流水线的加速比

完成同样一批任务，**不使用**流水线所用的时间与**使用**流水线所用的时间之比称为**流水线的加速比。**

计算流水线加速比的基本公式如下：
$$
S = \frac{不使用流水线执行时间}{使用流水线执行时间}
$$
例：若指令流水线把一条指令分为取指、分析和执行三部分，且三部分点的时间分别是取指2ns，分析2ns，执行1ns。那么，未使用流水线周期是多少？100条指令全部执行完毕需要的时间是多少？流水线的加速比是多少？

```
未使用流水线周期：2+2+1 = 5ns
100条指令全部执行完毕需要的时间：100*5 = 500ns
流水线的加速比：S = 500 / 203
```

### 流水线的效率

流水线的效率是指流水线的设备利用率。在时空图上，流水线的效率定义为$n$个任务占用的时空区与$k$个流水段总的时空区之比。

①计算流水线效率的公式为：
$$
E = \frac{n个任务占用的时空区}{k个流水段的总的时空区} = \frac{T_0}{kT_k}
$$
![image-20210318145817801](https://cdn.jsdelivr.net/gh/TheFoxFairy/ImgStg/202201112000315.png)

假设要完成的任务分成四部分：S1，S2，S3，S4，完成时间分别为$\Delta t$，$\Delta t$，$\Delta t$，$\Delta 3t$，可知流水线周期为$\Delta 3t$。

```
一条指令执行时间：△t + △t + △t + 3△t = 6△t 
这里一共4条指令： 6△t * 4 

E =  6△t * 4 / 15△t *4
```

②**从图中的面积理解：流水线的效率等于阴影所占方块与图中所有方块的比值**

### Cache的基本概念

* **Cache的功能：**提高CPU数据输入输出的速率，突破冯$\cdot$诺依曼瓶颈，即**CPU与存储系统间数据传送带宽限制**。

* 在计算机的存储系统体系中，Cache是访问速度**最快**的层次。
* 使用Cache改善系统性能的依据是**程序的局部性原理**。

如果以$h$代表对Cache的访问命中率，$t_1$表示Cache的周期时间，$t_2$表示主存储器周期时间，以读操作为例，使用”Cache+主存储器“的系统的平均周期为$t_3$，则：
$$
t_3 = h\times t_1 + (1-h)\times t_2
$$
其中，$(1-h)$又称为失效率（未命中率）。

### 时间局部性和空间局部性

* 时间局部性

例：

```
int i,s=0;
for(i=0;i<1000;i++)
	for(j=0;j<1000;j++)
		s+=j;
printf("%d",s);
```

**第一行语句只需运行一次，后面的循环语句中`s+=j`需要被调用10000000次，只需从Cache里面不断取出即可，不需要从内存里取，大大提高了运行速度。**

* 空间局部性：程序往往有访问一个聚集空间的数据块的倾向，参见数组的访问；

* 工作集理论：工作集是进程运行时被频繁访问的页面集合。

### 随机存储器和只读存储器

**主存分为随机存储器和只读存储器。**

* 随机存取存储器（RAM)，也叫主存

  * DRAM（Dynamic RAM，动态RAM）- SDRAM

  只读存储器（ROM）：

* * MROM（Mask ROM，掩模式ROM）

## 校验码

### 校验码概念

#### 什么是检错和纠错？

检错：检查错误。

纠错：检查错误并且纠正错误。

#### 什么是码距？

一个编码系统的码距是整个编码系统中任意（所有）两个码字的最小距离。

例：

若用1位长度的二进制编码。若A=1，B=0。这样A，B之间的最小码距为1；

若用2位长度的二进制编码。若A=11，B=00。这样A，B之间的最小码距为2；

若用3位长度的二进制编码。若A=111，B=000。这样A，B之间的最小码距为3；

#### 码距与检错、纠错有何关系？

* 在一个码组内为了检错$e$个误码，要求最小码距$d$应该满足：$d\ge e+1$

* 在一个码组内为了纠正$t$个误码，要求最小码距$d$应该满足：$d\ge 2t+1$
